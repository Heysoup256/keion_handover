# 軽音部 引き継ぎ資料アクセス認証システム

このプロジェクトは、軽音部の後輩に対して、Google Drive の共有フォルダや Google Sites へのアクセス権限を安全に付与するためのシステムです。

**設計の特徴:**
- 完全 GAS (Google Apps Script) 完結型です。外部サーバーや GitHub Pages は使用しません。
- ユーザーが直接アクセスする URL は GAS の推測不可能な URL のみです。
- クライアントID等の機密情報はすべて GAS のバックエンドに秘匿されるため、コードを公開しても安全です。

---

## セットアップ手順

### Step 1: GAS プロジェクトの作成とコード配置

1. Google Drive 上で新規スプレッドシートを開き、メニューから **「拡張機能」 > 「Apps Script」** をクリックします。
2. エディタが開いたら、リポジトリ内の `Code.gs` と `Index.html` のコードをコピーして貼り付けます。

### Step 2: 初回デプロイ (URL の取得)

OAuth 認証の設定をするためには、先に「GAS アプリの URL」を取得する必要があります。

1. 右上の **「デプロイ」 > 「新しいデプロイ」** をクリックします。
2. アプリの種類で **「ウェブアプリ」** を選択します。
   - 説明: `v1` など
   - 実行ユーザー: **「自分」**
   - アクセスできるユーザー: **「全員」**
3. 「デプロイ」をクリックし、表示された **「ウェブアプリの URL」をコピー** して控えておきます。
   *(例: `https://script.google.com/macros/s/AKfy.../exec`)*

### Step 3: Google Cloud Console で OAuth 認証情報の取得

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスします。
2. 対象プロジェクト（または新規プロジェクト）を選び、**「API とサービス」 > 「認証情報」** を開きます。
   *(※初めての方は「OAuth 同意画面」を「外部」で作成してください)*
3. **「＋認証情報を作成」 > 「OAuth クライアント ID」** を選択します。
   - アプリケーションの種類: **「ウェブ アプリケーション」**
   - 名前: `軽音部 認証システム` 等
   - **承認済みのリダイレクト URI**: ここに先ほど **Step 2 で取得した GAS のウェブアプリ URL** をそのまま貼り付けます。
     *(※承認済みの JavaScript 生成元 は空欄で構いません)*
4. 「作成」をクリックすると、作成完了画面に以下の2つが表示されます。これを両方コピーして控えます。
   - **クライアント ID**
   - **クライアント シークレット**

### Step 4: GAS に環境変数を設定 (Script Properties)

GAS エディタに戻り、左端の歯車アイコン ⚙️（プロジェクトの設定）を開き、一番下の「スクリプト プロパティ」に以下の値を登録します。

| プロパティ名 | 設定する値 |
| :--- | :--- |
| **`GOOGLE_CLIENT_ID`** | Step 3 で取得した **クライアント ID** |
| **`GOOGLE_CLIENT_SECRET`**| Step 3 で取得した **クライアント シークレット** |
| **`PASSWORD_HASH`** | 後輩に教えるパスワードの SHA-256 ハッシュ値。※取得方法は後述 |
| **`DRIVE_IDS`** | 権限を付与したい Drive のファイル/フォルダ ID（カンマ区切り） |
| **`SITES_FILE_ID`** | 権限を付与したい Google Sites のファイル ID |
| `ALLOWED_DOMAINS` | (任意) 許可するメールドメイン（例: `gmail.com, example.ac.jp`） |

**※ パスワードハッシュの取得方法**
1. エディタ上部の関数選択プルダウンから `generatePasswordHash` を選び、「実行」をクリックします。（※初回はGoogleの承認ダイアログが出ます）
2. 実行ログに表示されるハッシュ値を `PASSWORD_HASH` に設定してください。（※自分のパスワードにしたい場合は `Code.gs` 下部のコードを書き換えて実行してください）。

### Step 5: テストと利用開始

1. すべての準備が完了です。Step 2 で取得した Web アプリの URL（`/exec`）にブラウザでアクセスします。
2. 「Google でログイン」ボタンを押すと、Google のアカウント選択画面が出ます。
3. ログイン後、自動的にパスワード入力画面に戻ってきます。
4. 正しいパスワードを入力すると、権限が付与されます。

> ⚠️ **注意:**
> GAS エディタの「テスト デプロイ」（URLが `/dev` で終わるもの）でテストを行う場合、GCP のリダイレクト URI に `/dev` の URL も登録していないと `redirect_uri_mismatch` エラーになります。**テスト時も必ず `/exec` URL を使用することをお勧めします。**

## コードを修正した場合のデプロイ更新

GAS のコードを修正した場合は、保存しただけでは `/exec` URL には反映されません。
**必ず「デプロイ」>「デプロイを管理」から、既存のデプロイを編集して新バージョンとして公開し直してください。**

1. デプロイ > デプロイを管理 を開く
2. 既存の項目（例: Web アプリ）をクリックし、右上の ✏️ (編集) アイコンを押す。
3. バージョンを「新バージョン」に変更し、保存。
これにより、URL を変更することなく新しいコードを反映できます。

## 開発と保守 (clasp)

このリポジトリは GitHub 等での公開を前提に設計されています。
機密情報はすべてプロパティに保存するため、このコード群をパブリックリポジトリに置いても安全です。
`clasp` を使ってローカル開発環境から直接 GAS に Push することも可能です。
